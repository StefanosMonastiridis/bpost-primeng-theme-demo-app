Semantic colours

Semantic colours convey meaning to the user. For custom components use the following variables

Naming convention

Semantic colours follow the below naming convention:

  • prefix: color. All variable names of color start with the word 'color'
  • color type denominator: primary, secondary, info, validation, warning, error. Declares the semantics of each colour.
  • contrast denominator: low, default, high, higher. Declares the contrast of the color with a (white) background.
  • suffix: light, dark. Declares the colour theme

Neutral colours

Neutral colours are based on two grey variants and are classified according to their use:

  • Surfaces
  • Text
  • Outline

Naming convention

Semantic colours follow the below naming convention:

  • prefix: color. All variable names of color start with the word 'color'
  • colour use denominator: surface, outline, text. Declares the use of each colour.
  • for surface and outline colours, contrast denominator: lowest, low, default, high, higher, highest. Declares the contrast of the colour with a (white) background.
  • for text colours, contrast denominator: primary, secondary, disabled . Declares the priority of text, with primary the default
  • for text colours, inverse denominator: inverse
  • suffix: light, dark. Declares the colour theme
