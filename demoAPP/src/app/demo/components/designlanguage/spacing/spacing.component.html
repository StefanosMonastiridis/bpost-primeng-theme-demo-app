Scale

BPOST Design System uses a uniform scale with sizes from 0.25 to 3rem, represented by the following variables

$scale-xx-small: 0.25rem;

$scale-x-small: 0.5rem;

$scale-small: 0.75rem;

$scale-medium: 1rem;

$scale-large: 1.5rem;

$scale-x-large: 2rem;

$scale-xx-large: 2.5rem;

$scale-xxx-large: 3rem;

Implementation

Scale variables are assigned to different variable sets named according to use:

  • spacing: used for padding, margin and gaps.
  • size: used for width and height of components and containers.
  • radius: used for rounded corners of components and containers.

These variable sets are used to theme the native Prime NG components and in Figma

Custom components

For custom component templates BPOST design system supports the following options.

  • Use of our custom variable sets. The use of our variables ensures future-proof coding since it ensures that any change in scale will be immediately propagated in your custom components
  • Use of PrimeFlex classes. Most of the used scale has an equivalent utility class in PrimeFlex.
  • Use of Tailwind CSS utility classes. With the latest update Figma translates component structures in html with reference to Tailwind CSS utility classes. Please contact the Design System team for demo and pilot implementation. Requires to install Tailwind CSS
